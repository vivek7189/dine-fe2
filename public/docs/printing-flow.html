DineOpen Printing Flow

Internal guide — How printing works on Web (Browser) and Tauri (Desktop App)

1. Two Platforms, Same Logic

The print trigger logic (flags, settings, button clicks) is identical on both platforms. Only the print execution differs.

Web Browser

  • Opens a hidden window with print HTML
  • Calls window.print()
  • System print dialog always appears
  • User clicks "Print" in dialog

Tauri Desktop App

  • Calls Rust print_document command
  • Uses lp (Mac) or PowerShell (Windows)
  • Silent print if default printer is set
  • Falls back to browser dialog if no printer set

2. The 4 Print Buttons

There are 4 buttons on the billing screen. Each one's print behavior is controlled by a setting flag.

Place Order (KOT) Print optional

Sends order to kitchen display/tickets.

Prints KOT only if autoPrintOnPlaceOrder = ON

Default: OFF — no print happens

KOT & Print Always prints

Sends order to kitchen + always prints KOT.

Button visibility: enableKOTAndPrint

Default: Hidden (button not shown)

Complete Billing Print optional

Completes billing + processes payment.

Prints bill only if autoPrintOnCompleteBilling = ON

Default: OFF — no print happens

Bill & Print Always prints

Completes billing + always prints bill.

Button visibility: enableSaveAndPrint

Default: Hidden (button not shown)

6. All Print Settings (Admin → Print Settings)

Button Visibility

enableKOTAndPrint
Shows/hides the "KOT & Print" button. When clicked, always prints KOT.
enableSaveAndPrint
Shows/hides the "Bill & Print" button. When clicked, always prints bill.
manualPrintEnabled
Shows "Print" button on order summary for manual reprinting.

Auto-Print on Action

autoPrintOnPlaceOrder
When ON, "Place Order" button auto-prints KOT. No need for separate "KOT & Print" button.
autoPrintOnCompleteBilling
When ON, "Complete Billing" button auto-prints bill. No need for separate "Bill & Print" button.

Pusher / External App (Advanced)

kotPrinterEnabled
Enables dine-kot-printer external app workflow (mainly for Web).
autoPrintOnKOT
Pusher-triggered KOT print. For external printer app or multi-terminal Tauri setup.
autoPrintOnBilling
Pusher-triggered bill print. For external printer app or multi-terminal Tauri setup.
usePusherForKOT
Master switch for Pusher-based auto-print. Must be ON for the above two to work.

Display Settings (Not Printing)

showKOTSummaryAfterOrder
Show KOT summary card on dashboard after placing an order.
showBillSummaryAfterBilling
Show bill summary card on dashboard after completing billing.

Print Decision Tree

Button Click
  |
  |-- "Place Order (KOT)" --> placeOrder()
  |     |__ After success --> useEffect checks:
  |           if (__autoPrintKOT || autoPrintOnPlaceOrder) --> PRINT KOT
  |
  |-- "KOT & Print" --> sets __autoPrintKOT = true --> placeOrder()
  |     |__ After success --> useEffect checks:
  |           if (__autoPrintKOT || autoPrintOnPlaceOrder) --> PRINT KOT (always true)
  |
  |-- "Complete Billing" --> processOrder()
  |     |__ After success --> useEffect checks:
  |           if (__autoPrintBill || autoPrintOnCompleteBilling) --> PRINT BILL
  |
  |-- "Bill & Print" --> sets __autoPrintBill = true --> processOrder()
        |__ After success --> useEffect checks:
              if (__autoPrintBill || autoPrintOnCompleteBilling) --> PRINT BILL (always true)


Print Execution:
  |
  |-- supportsNativeAutoPrint()?
  |     |-- YES (Tauri) --> invoke('print_document') --> Rust --> lp/PowerShell --> Silent Print
  |     |-- NO  (Web)   --> window.open() --> window.print() --> System Dialog

9. Troubleshooting

Print dialog showing on Tauri desktop (not silent)

No default printer is set in the Tauri app.

Fix: Go to Admin → Print Settings → Click "Scan Printers" → Select your thermal printer → Click "Test Print" to verify silent printing works.

Print not happening on "Place Order" or "Complete Billing"

The auto-print settings are OFF (default).

Fix: Go to Admin → Print Settings → Turn ON "Auto-Print on Place Order" and/or "Auto-Print on Complete Billing".

"KOT & Print" or "Bill & Print" button not visible

These buttons are hidden by default.

Fix: Go to Admin → Print Settings → Turn ON "KOT & Print Button" or "Bill & Print Button".

Order History not printing silently on Tauri

Fixed in latest update. Order History now uses native printDocument() on Tauri.

Verify: Make sure your app is updated to the latest version and a default printer is set.
