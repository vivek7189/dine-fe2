# GCS Release Process — DineOpen POS (Electron)

## Overview

The Electron desktop app (`.dmg` for Mac, `.exe` for Windows) auto-updates from a public GCS bucket (`dineopen-releases`). When a new version is uploaded to the bucket, running apps detect it and update silently in the background.

## GCS Bucket

- **Bucket name:** `dineopen-releases`
- **Public URL:** `https://storage.googleapis.com/dineopen-releases/`
- **Access:** Public read (anyone can download), write requires `gsutil` auth

## How auto-update works

1. App starts → waits 5 seconds → checks `latest-mac.yml` (Mac) or `latest.yml` (Windows) from the bucket
2. Compares the version in the yml file with the current app version
3. If newer version found → downloads the installer silently in the background
4. On next app quit → installs the update automatically
5. Users can also manually check from **Admin/Settings → Automatic Updates → Check for Updates**

## Files in the bucket

```
dineopen-releases/
├── DineOpen-POS-{version}-setup.exe        # Windows installer
├── DineOpen-POS-{version}-setup.exe.blockmap
├── latest.yml                               # Windows version metadata
├── DineOpen-POS-{version}-arm64.dmg         # macOS installer
├── DineOpen-POS-{version}-arm64.dmg.blockmap
└── latest-mac.yml                           # macOS version metadata
```

The `.yml` files are auto-generated by electron-builder during build. They contain the version number, filename, sha512 hash, and file size. **Without the yml files, auto-update will not work.**

## Release workflow

### Step 1: Bump version

Update `version` in `package.json` before building:

```json
"version": "1.8.0"
```

Commit and push a git tag:

```bash
git tag v1.8.0
git push origin v1.8.0
```

### Step 2: GitHub Actions builds Windows .exe

Pushing the tag automatically triggers GitHub Actions to build the Windows `.exe`.

1. Go to **GitHub → Actions → "Build Electron App (Windows)"**
2. Wait for the build to complete
3. Download the artifact (`windows-release-v1.8.0`) from the workflow run
4. Extract and copy these 3 files into the `gcs-releases/` folder:
   - `DineOpen-POS-{version}-setup.exe`
   - `DineOpen-POS-{version}-setup.exe.blockmap`
   - `latest.yml`

### Step 3: Build Mac .dmg + Upload everything to GCS

On your Mac, run:

```bash
npm run release:gcs
```

This single command:
- Builds the Mac `.dmg` locally
- Copies `.dmg`, `.blockmap`, and `latest-mac.yml` to `gcs-releases/`
- Validates that both Windows and Mac files are present
- Uploads everything in `gcs-releases/` to the GCS bucket

### Alternative: Upload only (no Mac build)

If you already have the `.dmg` in `gcs-releases/` and just want to upload:

```bash
npm run release:gcs:upload
```

## npm scripts

| Command | Description |
|---|---|
| `npm run release:gcs` | Build Mac `.dmg` locally + upload all files in `gcs-releases/` to GCS |
| `npm run release:gcs:upload` | Upload only (no build) — use when all files are already in `gcs-releases/` |
| `npm run electron:build` | Build locally without uploading |

## Config files

| File | Purpose |
|---|---|
| `electron-builder.yml` | Build config, publish provider set to `gcs` + `dineopen-releases` |
| `electron/main.js` | `autoUpdater.setFeedURL` points to GCS bucket |
| `scripts/release-gcs.js` | Build + upload script |
| `src/utils/autoUpdater.js` | Frontend auto-update utility |
| `.github/workflows/build-electron.yml` | GitHub Actions workflow for Windows .exe build |

## Troubleshooting

**Auto-update not working?**
- Verify the yml files exist in the bucket: `gsutil ls gs://dineopen-releases/`
- Check the public URL loads: `https://storage.googleapis.com/dineopen-releases/latest-mac.yml`
- Ensure the version in `package.json` was bumped before building

**gsutil auth issues?**
- Run `gcloud auth login` to authenticate
- Or set `GOOGLE_APPLICATION_CREDENTIALS=/path/to/service-account.json`
